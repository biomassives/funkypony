Create your Supabase project

In progress
▶
Open Supabase → New Project
  • ✓
    Sign up or log in at supabase.com
  • ✓
    Create a new project — choose a region close to your users
  • ✓
    Note your Project URL — Settings → API → Project URL
  • ✓
    Note your anon public key — Settings → API → Project API Keys
  • ✓
    Note your service_role key — same page, keep this secret

Run migrations

Once your project is created, run this from the repo root to push all schema migrations:

npx supabase link --project-ref YOUR_PROJECT_REF
npx supabase db push

Or paste each file from supabase/migrations/ into the Supabase Dashboard SQL editor in filename order.

5
Set Supabase edge function secrets
So the deployed functions can send email and make calls
Pending
▶

Supabase edge functions read secrets separately from Vercel env vars. Set them once via CLI:

npx supabase secrets set \
  MAILGUN_API_KEY="your_key" \
  MAILGUN_DOMAIN="mg.yourdomain.com" \
  NOTIFY_FROM_EMAIL="notify@mg.yourdomain.com" \
  TWILIO_ACCOUNT_SID="ACxxxx" \
  TWILIO_AUTH_TOKEN="xxxx" \
  TWILIO_FROM_NUMBER="+13035550001"

Or set them in the Supabase Dashboard → Edge Functions → function name → Secrets.

  • ✓
    All secrets set for mts function (email)
  • ✓
    Secrets set for daily-digest, mailgun-webhook
  • ✓
    Secrets set for daily-call, twiml-gather (if using Twilio)

You're live on Vercel ▲

Open your Vercel deployment URL, complete the pantry setup wizard, and send your first invite. Then set your Supabase Site URL to match your deployment domain.

Heroku Postgres — Setup
How this path works: Heroku provides the PostgreSQL database via an add-on attached to a Heroku app. Supabase handles authentication and edge functions. Simplest option if you already have a Heroku account.
1
Create Heroku app + add Postgres
~5 minutes — Essential plan starts at $5/mo
In progress
▶
Create Heroku AppHeroku Postgres add-on
  • ✓
    Sign in at dashboard.heroku.com
  • ✓
    Create a new app (the app itself won't run anything — it's just a container for the database)
  • ✓
    Go to Resources → search for Heroku Postgres → attach it (Essential-0 plan)
  • ✓
    Go to Settings → Config Vars → copy the DATABASE_URL value
The DATABASE_URL from Heroku uses the postgres:// scheme. You may need to replace it with postgresql:// depending on your driver.

Run migrations against Heroku Postgres

psql YOUR_DATABASE_URL -f supabase/migrations/20260218_site_messages.sql
psql YOUR_DATABASE_URL -f supabase/migrations/20260222_profile_media.sql
# ... run each migration file in order
2
Create Supabase project (auth + functions)
Supabase manages sessions and edge functions; Heroku holds the data
Pending
▶
Open Supabase → New Project
  • ✓
    Create Supabase project, note URL, anon key, and service_role key
  • ✓
    Under Settings → Database → connect Supabase to your external Heroku database (Supabase supports external Postgres via the "connect your own database" option on Pro plans)
Connecting an external database requires a Supabase Pro plan ($25/mo). On the free plan, use Supabase's own database and export/import from Heroku, or skip Supabase and use a different auth provider.
